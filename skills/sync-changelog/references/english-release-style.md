# English Release Style

将中文版本日志转成英文时，遵循以下风格：

1. 面向英文母语用户，自然表达，不逐字翻译。
2. summary 控制在 1 句，简洁清晰，突出用户可感知变化。
3. items 使用短句，动词开头优先（例如："Improve...", "Fix...", "Add..."）。
4. 保留技术准确性，不夸大功能，不引入原文没有的信息。
5. 保留原分类结构（feat/fix/docs/chore 等），仅改写文本。

## 示例（仅风格参考）

中文："优化 AI 对话底部配置状态展示"

推荐英文："Improve the configuration status display in the AI chat footer."
